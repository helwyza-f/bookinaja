# Bookinaja Mobile Screen Priority Checklist For Stitch

Use this as the execution checklist while generating designs in Stitch.

## Phase 0 - Setup

- [ ] Read [bookinaja-mobile-stitch-design-brief.md](/D:/projects/bookinaja/docs/bookinaja-mobile-stitch-design-brief.md)
- [ ] Use [stitch-master-prompt.md](/D:/projects/bookinaja/docs/stitch-master-prompt.md) as the global prompt
- [ ] Lock one shared visual language before designing individual screens
- [ ] Lock customer and admin brand relationship before designing flows

Done when:

- [ ] Bookinaja has a clear visual identity
- [ ] The app no longer feels like a generic operational product
- [ ] Customer and admin feel related but distinct

## Phase 1 - Foundations

- [ ] Color system
- [ ] Surface system
- [ ] Typography system
- [ ] Radius system
- [ ] Elevation and depth system
- [ ] Icon style
- [ ] Live status style
- [ ] Motion principles
- [ ] Toast and feedback system
- [ ] Centered modal system
- [ ] Bottom sheet system

Done when:

- [ ] Foundations feel premium
- [ ] Surfaces feel alive, not flat
- [ ] Primary actions feel unmistakable
- [ ] Statuses are readable at a glance

## Phase 2 - App Shell And Navigation

- [ ] Root shell
- [ ] Customer shell
- [ ] Admin shell
- [ ] Bottom nav
- [ ] Scroll behavior
- [ ] Floating dock behavior
- [ ] Header behavior
- [ ] Search field behavior
- [ ] Segmented tabs behavior

Done when:

- [ ] All pages feel part of one product
- [ ] Nav feels polished
- [ ] Content never collides with tab bar or safe areas

## Phase 3 - Pattern Library

- [ ] Dashboard pattern
- [ ] List pattern
- [ ] Detail and action pattern
- [ ] Form and create pattern
- [ ] Picker and sheet pattern
- [ ] Cart and selection pattern
- [ ] State module pattern

Done when:

- [ ] These patterns can be reused across most screens
- [ ] Each pattern has clear hierarchy
- [ ] Action-first behavior is obvious

## Phase 4 - Customer Core

Priority A:

- [ ] Entry / landing
- [ ] Discovery
- [ ] Tenant storefront
- [ ] Resource detail and booking builder
- [ ] Checkout / payment

Priority B:

- [ ] Booking detail
- [ ] Booking live
- [ ] Orders detail
- [ ] Orders payment

Priority C:

- [ ] Login
- [ ] Register
- [ ] Google claim
- [ ] Profile
- [ ] History
- [ ] Explore

Done when:

- [ ] Booking journey feels exciting and easy
- [ ] Live journey feels active and premium
- [ ] Payment feels trustworthy and smooth

## Phase 5 - Admin Core

Priority A:

- [ ] Admin login
- [ ] Dashboard
- [ ] Bookings list
- [ ] Booking detail / action center
- [ ] Operations command center

Priority B:

- [ ] Direct sale / POS create
- [ ] Direct sale / POS detail
- [ ] Booking creation
- [ ] Customers list
- [ ] Customer detail

Priority C:

- [ ] Workspace
- [ ] Business profile

Done when:

- [ ] Admin feels like a premium command app
- [ ] The most important action is obvious in every state
- [ ] Live and payment states feel active and responsive

## Phase 6 - Admin Management Screens

- [ ] Resources list
- [ ] Resource detail / manage
- [ ] Menu management
- [ ] Expenses

Done when:

- [ ] CRUD screens still feel premium
- [ ] Admin management does not regress into generic forms
- [ ] Density feels efficient, not cramped

## Phase 7 - State Pass

For every high-priority screen:

- [ ] loading
- [ ] empty
- [ ] error
- [ ] success
- [ ] partial progress
- [ ] active/live
- [ ] completed
- [ ] blocked

Done when:

- [ ] State changes are visually meaningful
- [ ] The product never feels dead or silent

## Phase 8 - Motion Pass

- [ ] Entry transitions
- [ ] Tab transitions
- [ ] Sheet motion
- [ ] Modal motion
- [ ] Toast motion
- [ ] Refresh motion
- [ ] Success microfeedback
- [ ] Live pulse behavior

Done when:

- [ ] Interactions feel smooth
- [ ] Motion supports clarity
- [ ] Motion adds character without noise

## Phase 9 - Polish Pass

- [ ] Reduce verbose copy
- [ ] Replace weak text labels with better icon + layout cues
- [ ] Tighten spacing inconsistencies
- [ ] Improve grid rhythm
- [ ] Improve hierarchy in sections that still feel flat
- [ ] Make hero surfaces more memorable
- [ ] Improve visual feedback in live and payment moments

Done when:

- [ ] The app feels expensive
- [ ] The app feels rememberable
- [ ] The app feels like Bookinaja

## Shipping Bar

Do not consider the design done until:

- [ ] it feels useful
- [ ] it feels smooth
- [ ] it feels delightful
- [ ] it feels alive
- [ ] it feels premium
- [ ] it feels emotionally intentional
- [ ] it feels talk-worthy
